# 前言
大约前年10月份这样，我萌生了组装一台服务器的念头。因为有学习linux的想法，但是缺少一个可以随便折腾并且编译迅速的平台。当时想过使用EPYC，但是最终还是选择了成熟的双路E5（配置可能之后会贴上来）。然后按照学长的推荐使用了proxmox系统。然后问题就随之而来了。
# 网络无法访问
由于proxmox的网络通过一个vm0什么的虚拟设备管理，然后这个会绑定另外一个ensp5啊ensp6啊的实际网络硬件，通过配置文件来管理。这就导致如果你插拔PCIE上的设备，ensp这个编号就会改变，然后配置文件就会错误连接到一个没网（而且基本上是100%错误链接）的设备上。然后就会导致实际网卡被禁用，没插的网卡反而激活这种神秘问题。然后你就需要手动修改配置文件，使用ip link show查清楚哪个是能用的，哪个不能用。然后修改底下一个config文件改为正确的硬件网口。
# 傲腾！！
这个傲腾更是神秘，因为恰逢内存大涨价，带着储存一起起飞，所以我就去咸鱼淘了个神秘傲腾NVME 固态，然后更神秘的是这个玩意偶尔能识别偶尔不能识别，导致一开始装在上面的主系统proxmox经常死机。后面转为承载虚拟机，也得反复重启才能识别成功。
# tailscale！
当然，这台服务器不是连着显示器用的。我更喜欢用tailscale远程连接去进行各种编程与开mc服务器等操作。可是由于tailscale默认使用最后一次连接上的连接方式进行连接，然后我从家里回来之后就因为各种原因没有碰过这个服务器，导致现在重新开起来默认使用relay连接，但是这个relay是走derp的，这个derp又是在腾讯云上，是需要一个域名和证书的。然后这个证书是自签发，又会过期，所以这就导致derp看起来可用但是实际上出了问题。这样一来我就算能打洞直连也必须走这个已经挂掉的中继，也就是连不上了。后面不得不删掉derpmap配置，然后改为走最新的relay功能。
此外还有校园网经常出问题需要重新认证等等....
